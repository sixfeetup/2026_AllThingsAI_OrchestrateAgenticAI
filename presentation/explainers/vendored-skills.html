Vendored Skills

The tools, skills, and scripts that built the Agent of Legends presentation — vendored into the repo with documentation of their historical importance. Five categories, 19 artifacts, one self-referential thesis.

1

Build Pipeline

The core compilation loop: markdown slides in, HTML deck out. These scripts were the first non-content artifacts created and were iterated on more than any other component.

build.py

Single-file Python slide compiler. Parses YAML frontmatter + markdown body from content/*.md, renders HTML slides, wraps them in template.html. Modified 6+ times across the project's life.

  • YAML frontmatter: type, image, image_layout, footer
  • ??? separator for speaker notes (Remark.js convention)
  • @@ directive scanning + stripping
  • Image path rewriting for output depth (../images/ → ../../images/)
  • Quote extraction into absolute-positioned .slide-quote
  • Blockquote splitting on blank > lines, link rendering inside quotes

check-changes.sh

MD5 hash-based change detection. Computes a hash of all watched files, compares to stored .watch-state. Exit 0 = changed, exit 1 = no changes. Makes the 90-second watch interval practical — most ticks are no-ops.

Makefile

Build orchestration: build, open, clean, deploy (gh-pages), serve, watch, list. The deploy target syncs referenced images and commits to the gh-pages branch. Uses uv run build.py.

Watch Loop

The continuous integration system. One command — /loop 90s /deck-watch — set the heartbeat for the entire development workflow.

/deck-watch

The core watcher skill. Five-step algorithm on every 90-second tick:

  • Check — run check-changes.sh; bail if hash unchanged
  • Read — load plan.md, README.md, quotes.md, all content/*.md
  • Process @@ — find inline directives, execute them, remove from source
  • Regenerate — compare plan structure vs existing slides, create/update/delete
  • Build — run make build, report result

/render-deck

Manual build-and-open. Wraps make open. Used constantly for one-off rebuilds and quick previews. Documents slide markdown format, frontmatter fields, body syntax.

/loop (bundled)

Built-in Claude Code skill. Schedules recurring prompts via CronCreate. Usage: /loop 90s /deck-watch. The timer that makes the watch loop possible. Not vendored — ships with Claude Code.

The @@ Directive Innovation

The most distinctive feature. The author leaves inline instructions in any watched file:

## Testing Strategy
@@ add a slide about property-based testing
- Unit tests for core logic

On the next tick, the watcher finds the @@ line, interprets it, creates or modifies the appropriate slide, and removes the directive. Every source file becomes a two-way communication channel between author and AI.

Parallel Dispatch

Two complementary orchestrators for fanning work across independent AI agents. The project demonstrates its own thesis: using agents to build a talk about agents.

/c^2 (Claude²)

Agent-tool-native dispatch. Lightweight, no daemon. Workers run in isolated git worktrees, commit to branches, create PRs.

  • swarm — fan out 2-10 parallel workers
  • work — dispatch a single task
  • status / gather — track progress
  • msg — send course corrections
  • Multi-repo dispatch via registry

/mc (MultiClaude)

Daemon-based orchestrator. Persistent binary manages agent lifecycles, worktrees, messaging, PR workflows.

  • swarm — parallel research workers
  • attach / logs — live observability
  • Survives terminal disconnects
  • Long-running research tasks
  • Worker naming (happy-elephant, zealous-rabbit)

Content Generation

Tools for generating all non-text assets — images, HTML visualizations, demo data, and reference material downloads.

/visual-explainer

Generates self-contained HTML pages for technical diagrams, visualizations, and data tables. This very page was built with it. Supports Mermaid, CSS Grid, Chart.js, multiple aesthetic presets. Strong anti-slop guardrails.

Pages generated for this project:

  • system-overview.html Build system guide
  • playbook.html Operational playbook
  • demo-plan.html Live demo sequence
  • checklist-progress.html Readiness tracking
  • vendored-skills.html This page

/gen-image

Image generation via Gemini or Codex (DALL-E). Style hints, size control, backend selection. Used for slide backgrounds throughout.

/aix

1Password API key cache. Wraps op run to cache keys to /tmp/ai-keys.env. Required before any Gemini API calls.

generate_images.py

Batch Gemini image generator with 6 RPG-themed prompts. Skip-if-exists logic for safe re-runs.

dl.py

D&D archive PDF downloader. Scrapes reference PDFs to populate the demo scenario's OGL content library. Used to simulate the "OGL crisis" that the talk's narrative is built around — Lady Tasha Tiamata's fictional licensing emergency.

The Image Pipeline

/aix env → cache GEMINI_API_KEY → /gen-image "prompt" → Gemini API → images/*.png → referenced in slide frontmatter → build.py → path rewrite → deck.html

Session Management

Human-AI collaboration workflow: notifications, backgrounding, memory persistence, and session lifecycle. The connective tissue of the development process.

/say

macOS text-to-speech notifications. Used at every milestone: "Done. Deck deployed." "Swarm complete." Different voices for different event types — Zarvox for swarm completions. Enabled async collaboration: walk away, come back when you hear it.

/bg

Background job alias. The most-used prefix in the entire project. Nearly every command started with /bg to keep the UI responsive. Fire → do something else → come back when notified. The default interaction pattern.

/end-session

Born from this project. The user described it as a user story, then asked for it to become a skill. Reviews conversation for corrections, conventions, decisions. Writes to auto-memory. Has a UserPromptSubmit hook — just say "end session" and it triggers.

/finished

Transcript finalization. Locates session transcript, adds summary section, stamps completion time. Chains from /end-session for full session close-out.

Design Principles

  • Audio for async awareness — /say turns a text interface into one with ambient awareness. Listen, don't watch.
  • Background-first interaction — the human should never be blocked waiting for the agent.
  • Persistent memory — knowledge accumulates across sessions via /end-session auto-memory writes.
  • Natural language triggers — "end session" fires a hook, no slash command needed. Collaboration, not commands.
  • Memory vs transcript — /end-session captures what the agent should remember. /finished captures what the human should be able to read later.

Overview Map

How all 19 vendored artifacts connect. The build pipeline compiles, the watch loop drives, dispatch fans out, content generation creates assets, and session management keeps the human in the loop.

The Full Cycle

Session: /bg dispatch task → /say notify on completion
Watch: /loop 90s → /deck-watch detects change → processes @@
Build: check-changes.sh → build.py compiles → Makefile orchestrates
Swarm: /c^2 swarm fans out → N workers → N PRs → merge
Assets: /aix → /gen-image → /visual-explainer → self-contained HTML
Close: "end session" → /end-session → memory persisted → /finished
