Pre-bake vs. Live Strategy

Not everything should be live. Pre-baking slow or risky steps makes the demo more reliable without losing the narrative impact.

Element Strategy Rationale
PDF name extraction Pre-bake 30-60s per PDF is too slow for stage
DB cross-reference query Hybrid Pre-load DB, run query live (fast, dramatic)
Name generation Live Fast API call, impressive, low risk
Skill creation (/say) Live Quick paste-and-tweak, shows the concept
Contract ingest Live Fast, sets the scene for Demo 2
Visual explainers Pre-bake HTML generation too slow for live
Security report Hybrid Try live, fallback to pre-baked
/c^2 swarm dispatch Hybrid High risk, high reward — pre-bake ready
speckit output Pre-bake Too slow for 6-min window
multiclaude TDD Hybrid Try live, recording as fallback
ralph-orchestrator Pre-bake Recording only — too complex for 4 min

Demo 1: OGL Content Crisis

What the audience sees

The team uses an agent to search D&D PDFs for OGL-protected names, cross-references against L&LL's content database, reports damage, and generates replacement names. A skill is created on the fly to demonstrate how skills codify problem-solving.

Live sequence (6 min)

TimeActionWhat Audience Sees
0:00Set scene verbally"Tasha's call just came in."
0:30Open Claude Code, ls PDF archive18 D&D PDFs on screen
1:00Prompt: search PDFs for OGL namesAgent working, extracting entities
3:00Show damage reportNames found, affected content count
3:30Create /say skill on the flySkill file pasted and explained
4:30Generate replacement namesCreative alternatives appear
5:30Wrap and transition"The OGL crisis is quantified."

Key risk

PDF parsing speed. The full 18-file corpus (~170MB) could take 30-60s per file. Mitigation: Pre-extract names, use 2-3 PDFs live, show cached full results.

Demo 2: Contract/Security Review

What the audience sees

Ingest a contract PDF, generate visual explainers and checklists, run security skills, dispatch a swarm of agents (/c^2) using Codex and Gemini for adversarial review, and fire up speckit for specs.

Live sequence (6 min)

TimeActionWhat Audience Sees
0:00Transition"There's a trap in the contract."
0:30Ingest contract PDFClaude recognizes the document
1:00Generate explainer + checklistHTML output (or pre-baked)
2:00Run security skillsSecurity report output
2:30Launch /c^2 swarmCodex + Gemini dispatched
3:30Show visual explainer overlaysPre-baked HTML if needed
4:30Adversarial findingsMulti-model review results
5:00speckit demoSpec from findings
5:30Show worktrees + PRsEach agent on its own branch
6:00Wrap"Security baked in, not bolted on."

Key risk

Three external APIs chained together. Gemini and OpenAI quotas already hit during rehearsal. Mitigation: Pre-bake ALL visual explainer outputs. Consider making this demo 50% pre-baked by design.

Demo 3: Jetpacks / Working Together

What the audience sees

Specify a feature, launch multiclaude for TDD implementation, show ralph-orchestrator with hat-based loop-driven development. The shortest demo — focus on concept over terminal output.

Live sequence (4 min)

TimeActionWhat Audience Sees
0:00Show feature specBrief spec on screen
1:00Launch multiclaudeTDD cycle: tests first, then impl
2:30Switch to ralph-orchestratorHats prompt, loop iteration
3:30Show output: tests pass, PR createdClean implementation
4:00Wrap"The medium is the message."

Key risk

Tight time window (4 min). If multiclaude or ralph takes too long to produce visible output, the demo stalls. Mitigation: Pre-record both. The live attempt is a bonus; the recording is the plan.

The "Everything Fails" Plan

The talk's thesis is about managing non-deterministic systems. A gracefully-handled failure IS the demo. Lean into the meta-narrative: "This is exactly why we pre-bake, use fallbacks, and plan for graceful degradation. This is the orchestrator's dilemma in action."
PriorityAction
1Open pre-baked HTML visual explainers in browser tabs — these carry the narrative visually
2Narrate the demo: "Here's what you would see when we run this..."
3Show pre-baked output files (markdown reports, spec docs) as static content
4Play pre-recorded screen captures for Demo 3 (asciinema or video)
5Lean into the meta-narrative about managing failure in agentic systems

  • If WiFi is unreliable, switch to dedicated hotspot
