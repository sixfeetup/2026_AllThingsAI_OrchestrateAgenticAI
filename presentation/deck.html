The Broken Promise

"You hand a 20-page contract to the AI oracle. It misses the clause that matters. You correct it. It says 'Oh, you're right.'"
  • That's not a reviewer — that's a yes-man NPC
  • Same failure in code review, policy compliance, test planning
  • The party has been burned. Trust is low.

The Cleric

The Wizard

Developer. Reshapes reality with code but occasionally blows up the lab. Cares about shipping, quality, and not being on-call forever.

The Ranger

Ops / Security. Watches the perimeter, makes sure the ground doesn't collapse. Cares about reliability and audit trails.

The Warforged Artificer

Claude Code. Tireless construct. Loyal but literal. Only as good as its instructions and context.

02

Decision Frameworks

OODA Loop

Observe, Orient, Decide, Act. Every agentic interaction is an OODA cycle. The question: who does which step at each autonomy level?

Cynefin

Match autonomy level to problem complexity. Simple: automate. Complicated: multi-agent. Complex: human-in-the-loop. Chaotic: human takes the wheel.

  • At Level 1, the human does all four OODA steps
  • At Level 3, the agent runs its own loops — the human audits
  • The gap isn't intelligence — it's orchestration, context, and trust

II

The Levels

What does agentic AI actually look like?

04

Level 1 — The Apprentice

"You swing the sword. The AI sharpens it."
  • Human-orchestrated, step by step
  • Context engineering: chunking, targeted prompts, checklists
  • OODA: human does all four steps
  • Limitation: scales with your attention. You are the bottleneck.
05

Level 2 — The Party

"You plan the raid. The agents execute roles."
  • Multi-agent workflows: multiclaude, ralph, spec-kit
  • Decompose "review this" into a task graph of sub-jobs
  • Sandboxing: contained environments — mistakes don't cascade
  • Adversarial agents: pit one against another to find gaps
  • OODA shift: agents Observe/Orient/Decide. Human retains Act.
06

Orchestration Approaches

Conductor

Run multiple agents, bounce between windows. Good control, but decision fatigue for the human.

Relentless Loop

Set it and forget it. Low effort, but hard to audit. What have we built? What happened along the way?

Swarm

Many focused agents with isolated concerns. Great isolation, but needs higher-level orchestration.

Spec-Driven

Upfront specs force definition and create an audit trail. Best traceability, requires upfront work.

07

Level 3 — The Autonomous Guild

"You set the objective. The system runs the campaign."
  • Self-orchestrating agent networks
  • Trust earned through Level 2 reliability + audit trails
  • Cynefin: works for Simple/Complicated. Complex still needs human-in-the-loop.
  • The future is here, just unevenly distributed

III

The Dungeon

How to actually do this

09

Context Engineering

  • Chunk documents and tasks into focused units
  • Frame targeted prompts — tell the agent what to look for and what to ignore
  • Apply checklists with acceptance criteria per sub-task
  • Traceability: every sub-task has clear pass/fail criteria
  • Upstream: problem definition clarity is the leverage point
10

Trust as a Graduated System

  • Observe — watch the agent work, review every output
  • Verify — spot-check outputs against known-good baselines
  • Approve — review and sign off on agent-produced artifacts
  • Delegate — assign tasks, review only escalations
  • Autonomize — set objectives, audit outcomes
"Each level earned, not assumed."
11

Safety & Containment

  • Sandboxing: isolated execution, rollback, dry runs
  • Adversarial patterns: second agent challenges the first
  • Structured outputs: task graphs, checklists, diff-style reports
  • Readability and reasonability become primary structures of trust
"Security enables speed when baked in, not bolted on."

Demo: The Contract Review

Level 1: Single-Skill Checklist

  • Adding the rest of the party changes the game

Multi-Agent Decomposition

# Level 2: task graph decomposition async def review_contract(doc: Document) -> Report: clauses = await clause_extractor(doc) risks = await asyncio.gather( *[risk_scorer(c) for c in clauses] ) gaps = await completeness_validator(clauses, risks) review = await adversarial_check(clauses, risks, gaps) return await report_generator(risks, gaps, review)
  • Each agent has a focused concern and a contained scope
  • Adversarial check challenges the primary review output

Same Pattern, Different Domain

Contract Review

Extract clauses. Score risk. Validate completeness. Generate report.

Code Review

Extract functions. Analyze complexity. Generate test plan. Validate coverage.

Compliance Audit

Extract requirements. Map to controls. Gap analysis. Generate report.

  • Decompose. Specialize. Validate. Report.
  • The Cynefin domain determines autonomy. The OODA loop determines who does what.

The Playbook

  • Start at Level 1. Get context engineering right first.
  • Build trust incrementally. Level 2 before Level 3.
  • Use adversarial agents. Don't trust a single pass.
  • Write tools for vigilance, not just automation.
  • Security enables speed when baked into the workflow.
  • Specs are your audit trail.
